## Objective

Demonstrar el uso de hooks JavaScript (GraalJS) para consumir un endpoint de GitLab cuando un Entity Type (Task) cambie al estado "Closed".

### Purpose

Validar que cuando una Task transiciona al estado "Closed", se ejecuta un script JavaScript que:
1. Consume la API de GitLab para añadir una label
2. Usa el objeto `entity` para obtener el issueId
3. Añade la label "Bug :: Developer" al issue

### Output

- Script JavaScript de ejemplo para GitLab API
- Test unitario que verifica el script
- Configuración de variables de entorno
- Documentación del endpoint de GitLab

**GitLab API Reference:**
- Endpoint: `PUT /projects/:id/issues/:issue_iid`
- Parameter: `add_labels` (comma-separated label names)
- Authentication: `PRIVATE-TOKEN` header
- Documentation: https://docs.gitlab.com/api/issues/

  <files>bedomain/src/test/java/com/bedomain/service/GitLabHookTest.java</files>
  <action>
Crear script JavaScript de ejemplo que se ejecutará en el hook onEnter del estado "Closed":

```javascript
// Script: Añadir label "Bug :: Developer" al issue en GitLab cuando se cierra
// Este script se ejecuta al entrar al estado "Closed"

(function() {
    // Obtener configuración desde variables de entorno (accesibles via System.getenv)
    // En GraalJS sandbox, necesitamos pasar la config via contexto
    
    var gitlabUrl = entity._config.gitlabUrl;
    var gitlabToken = entity._config.gitlabToken;
    var projectId = entity._config.projectId;
    var issueIid = entity.issueId;
    
    if (!gitlabUrl || !gitlabToken || !projectId || !issueIid) {
        console.log("GitLab configuration missing, skipping label add");
        return entity;
    }
    
    // Construir URL para actualizar el issue
    var url = gitlabUrl + "/api/v4/projects/" + projectId + "/issues/" + issueIid;
    
    // Preparar body
    var body = JSON.stringify({
        add_labels: "Bug :: Developer"
    });
    
    // Headers requeridos
    var headers = {
        "PRIVATE-TOKEN": gitlabToken,
        "Content-Type": "application/json"
    };
    
    // Realizar request usando fetch nativo de GraalJS
    try {
        var response = fetch(url, {
            method: "PUT",
            headers: headers,
            body: body
        });
        
        if (response.ok) {
            console.log("Label added successfully to issue " + issueIid);
            entity._gitlabLabelAdded = true;
        } else {
            console.log("Failed to add label: " + response.status);
            entity._gitlabError = response.status;
        }
    } catch (e) {
        console.log("Error calling GitLab API: " + e.message);
        entity._gitlabError = e.message;
    }
    
    return entity;
})();
```

El script:
- Usa `entity._config` para recibir configuración (URL, token, projectId)
- Usa `entity.issueId` para obtener el issue IID
- Maneja errores apropiadamente
- Funciona con el timeout de 2 segundos del sandbox
