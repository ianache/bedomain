## Objective

Integrate JavaScript execution hooks into the state transition flow - execute onExit script from previous state, then onEnter script from new state.

### Purpose

Enables actual execution of user-defined scripts during state transitions with full audit trail.

### Output

- Modified StateTransitionService executing hooks
- Hook execution logged in StateHistory
- Proper error handling and rollback support

---

## Context

@bedomain/bedomain/src/main/java/com/bedomain/service/StateTransitionService.java

Current flow (from triggerTransition method):
1. Lookup EntityInstance
2. Get current state
3. Find valid transition
4. Update entity.currentState
5. Record history
6. Publish Kafka event

New flow with hooks:
1. Lookup EntityInstance
2. Get current state
3. Find valid transition
4. **Execute onExit script (from old state) - if exists**
5. **Execute onEnter script (from new state) - if exists**
6. Update entity.currentState
7. Record history (including hook execution info)
8. Publish Kafka event

## Tasks

### Task 1: Modify StateTransitionService to execute hooks

**Files:** `bedomain/src/main/java/com/bedomain/service/StateTransitionService.java`

**Action:**
Update `triggerTransition` method to:

1. **Add JavaScriptExecutor as dependency**:
```java
private final JavaScriptExecutor javascriptExecutor;
```

2. **Get state specs** for both old and new states:
```java
StateSpec fromStateSpec = stateMachine.getStates().stream()
    .filter(s -> s.getName().equals(fromState))
    .findFirst()
    .orElse(null);
    
StateSpec toStateSpec = stateMachine.getStates().stream()
    .filter(s -> s.getName().equals(newState))
    .findFirst()
    .orElse(null);
```

3. **Execute onExit script** (before changing state):
```java
if (fromStateSpec != null && fromStateSpec.getOnExitScript() != null) {
    entity.setAttributes(javascriptExecutor.execute(
        fromStateSpec.getOnExitScript(),
        entity.getAttributes(),
        entity.getId(),
        "onExit"
    ));
}
```

4. **Execute onEnter script** (after determining new state):
```java
if (toStateSpec != null && toStateSpec.getOnEnterScript() != null) {
    entity.setAttributes(javascriptExecutor.execute(
        toStateSpec.getOnEnterScript(),
        entity.getAttributes(),
        entity.getId(),
        "onEnter"
    ));
}
```

5. **Add hook execution info to history** (next task)

**Verify:**
```bash
cd bedomain && mvn compile -q
```

**Done:** StateTransitionService executes onExit and onEnter scripts during transitions

### Task 3: Add helper methods to JavaScriptExecutor

**Files:** `bedomain/src/main/java/com/bedomain/service/JavaScriptExecutor.java`

**Action:**
Add method with additional context:

```java
public Map<String, Object> execute(String script, Map<String, Object> context, 
                                    UUID entityId, String hookType) {
    // Add entityId and hookType to context as metadata
    Map<String, Object> fullContext = new HashMap<>(context);
    fullContext.put("_entityId", entityId.toString());
    fullContext.put("_hookType", hookType);
    fullContext.put("_timestamp", Instant.now().toString());
    
    return execute(script, fullContext);
}
```
