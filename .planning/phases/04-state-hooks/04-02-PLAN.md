## Objective

Create a secure JavaScript execution service using GraalJS with sandboxing, timeout, and error handling.

### Purpose

Provides the runtime capability to execute user-defined scripts with controlled access to entity context.

### Output

- JavaScriptExecutor service with sandboxed execution
- Configuration for resource limits
- Error handling for script failures

---

## Context

@bedomain/bedomain/pom.xml

From discovery (04-DISCOVERY.md):
- Use GraalJS with CONSTRAINED sandbox policy
- Resource limits: 2s CPU, 1000 statements, 64MB heap
- Scripts receive entity attributes as JavaScript object
- Scripts can modify attributes (returned to caller)

### Task 2: Create JavaScriptConfig

**Files:** `bedomain/src/main/java/com/bedomain/config/JavaScriptConfig.java`

**Action:**
Create configuration class for JavaScript engine:

```java
@Configuration
@ConfigurationProperties(prefix = "app.javascript")
public class JavaScriptConfig {
    private int maxCpuTimeSeconds = 2;
    private int maxStatements = 1000;
    private int maxHeapMemoryMb = 64;
    private boolean failOnError = true;
    private SandboxPolicy policy = SandboxPolicy.CONSTRAINED;
    
    // Getters and setters
}
```

Add to application.yml:
```yaml
app:
  javascript:
    max-cpu-time-seconds: 2
    max-statements: 1000
    max-heap-memory-mb: 64
    fail-on-error: true
```

**Verify:**
```bash
cd bedomain && mvn compile -q
```

**Done:** Configuration class loads JS settings from application.yml

### Task 3: Create JavaScriptExecutor service

**Files:** `bedomain/src/main/java/com/bedomain/service/JavaScriptExecutor.java`

**Action:**
Create service with:

1. **Execute method** accepting:
   - `script`: String with JavaScript code
   - `context`: Map<String, Object> - entity attributes
   - Returns: Modified context Map

2. **Sandboxing** using CONSTRAINED policy:
```java
Context context = Context.newBuilder("js")
    .sandbox(SandboxPolicy.CONSTRAINED)
    .option("sandbox.MaxCPUTime", config.getMaxCpuTimeSeconds() + "s")
    .option("sandbox.MaxStatements", String.valueOf(config.getMaxStatements()))
    .option("sandbox.MaxHeapMemory", config.getMaxHeapMemoryMb() + "MB")
    .out(new ByteArrayOutputStream())
    .err(new ByteArrayOutputStream())
    .build();
```

3. **Context injection**: Expose entity attributes as global `entity` object:
```java
context.getBindings("js").putMember("entity", contextAttributes);
```

4. **Result extraction**: After execution, extract modified `entity` object

5. **Error handling**: 
   - Wrap in try-catch
   - Log script errors
   - Return original context if failOnError=false
   - Throw ScriptExecutionException if failOnError=true

6. **Timeout**: Use sandbox.MaxCPUTime for CPU-bound timeout

**Verify:**
```bash
cd bedomain && mvn compile -q
```

**Done:** JavaScriptExecutor can execute scripts with context access
