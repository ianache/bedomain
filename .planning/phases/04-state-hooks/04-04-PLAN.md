## Feature Tests

### Test 1: JavaScriptExecutor basic execution

**Files:** `bedomain/src/test/java/com/bedomain/service/JavaScriptExecutorTest.java`

**Behavior:**
- Script can read entity attributes
- Script can modify entity attributes
- Modified attributes are returned to caller

```java
// Test: Script modifies attribute
Map<String, Object> context = Map.of("status", "pending", "amount", 100);
String script = "entity.amount = entity.amount * 1.1; entity.processed = true;";
Map<String, Object> result = executor.execute(script, context, entityId, "onEnter");

assertEquals(110.0, result.get("amount"));
assertEquals(true, result.get("processed"));
```

---

### Test 2: Sandbox enforces resource limits

**Behavior:**
- Infinite loop times out after maxCpuTimeSeconds
- Excess statements rejected after maxStatements

```java
// Test: Timeout
String infiniteLoop = "while(true) {}";
assertThrows(ScriptExecutionException.class, () -> 
    executor.execute(infiniteLoop, Map.of(), entityId, "onEnter"));
```

---

### Test 3: Script errors handled gracefully

**Behavior:**
- Syntax errors throw ScriptExecutionException
- Runtime errors logged, don't crash JVM
- failOnError=false returns original context

```java
// Test: Syntax error
String badScript = "entity.x = ;;;";
assertThrows(ScriptExecutionException.class, () ->
    executor.execute(badScript, context, entityId, "onEnter"));

// Test: failOnError=false
config.setFailOnError(false);
Map<String, Object> result = executor.execute(badScript, context, entityId, "onEnter");
assertEquals(context, result); // Original returned
```

### Test 4: StateTransitionService integrates hooks

**Files:** `bedomain/src/test/java/com/bedomain/service/StateTransitionServiceTest.java`

**Behavior:**
- onExit script executes when transitioning
- onEnter script executes when transitioning
- Attributes modified by hooks persist

```java
// Test: Hook modifies attribute during transition
when(toStateSpec.getOnEnterScript()).thenReturn("entity.approved = true;");
when(toStateSpec.getName()).thenReturn("APPROVED");

EntityInstance result = service.triggerTransition(entityId, "APPROVE");

verify(javascriptExecutor).execute(
    eq("entity.approved = true;"),
    any(), // original attributes
    eq(entityId),
    eq("onEnter")
);
assertEquals("APPROVED", result.getCurrentState());
```

---

### Test 5: Hook failure doesn't block (configurable)

**Behavior:**
- With failOnError=true, hook failure blocks transition
- With failOnError=false, transition proceeds

```java
// Test: failOnError=true blocks transition
when(javascriptExecutor.execute(any(), any(), any(), any()))
    .thenThrow(new ScriptExecutionException("Script error"));

assertThrows(ScriptExecutionException.class, () ->
    service.triggerTransition(entityId, "APPROVE"));
verify(entityInstanceRepository, never()).save(any());
```
