## Objective

Add onEnter and onExit script fields to StateSpec entity to allow storing JavaScript code that executes during state transitions.

### Purpose

Enables business logic customization at state level - validation, attribute transformation, notifications, etc.

### Output

- Updated StateSpec entity with script fields
- Updated DTOs
- CRUD support for scripts via existing StateMachine API

## Tasks

### Task 1: Add script fields to StateSpec entity

**Files:** `bedomain/src/main/java/com/bedomain/domain/entity/StateSpec.java`

**Action:**
Add two new columns to StateSpec entity:
- `onEnterScript` (TEXT, nullable): JavaScript code to execute when entering this state
- `onExitScript` (TEXT, nullable): JavaScript code to execute when exiting this state

Use `@Column(columnDefinition = "TEXT")` for MySQL compatibility.

**Verify:**
```bash
cd bedomain && mvn compile -q
```

**Done:** StateSpec entity has onEnterScript and onExitScript fields persisted to database
