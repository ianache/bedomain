<objective>
Implement CRUD APIs for entity instances. Entity instances are concrete records that conform to an entity type definition. They store attributes as JSON and reference their entity type.
</objective>

<context>
@.planning/phases/01-entity-foundation/01-CONTEXT.md
@.planning/phases/01-entity-foundation/01-RESEARCH.md

**Key constraints from CONTEXT.md:**
- Entity instance fields: id (GUID), type reference, attributes (JSON), creation date, created by
- Resource naming: /api/v1/entities
- Filtering by type and state (state deferred to Phase 2)
- Soft deletes (deleted flag)
- Audit fields: createdAt, createdBy, updatedAt, updatedBy

**Technology from RESEARCH.md:**
- Use JPA 2.2+ JSON column support with @JdbcTypeCode(SqlTypes.JSON)
- Validate entityTypeId exists before creating instance
- Consider attribute validation against property specifications (basic type checking)
</context>
