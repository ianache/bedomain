  <files>bedomain/src/main/java/com/bedomain/service/StateTransitionService.java</files>
  <action>
Create **StateTransitionService.java** following 02-RESEARCH.md patterns:

1. Lookup EntityInstance by ID
2. Get StateMachine config for the entity's EntityType
3. Use StateMachineFactory to create SM instance
4. Rehydrate SM from entity's currentState via StateMachineContext
5. Send event to SM - validates transition is allowed per SM rules
6. If rejected (sm.sendEvent returns false), throw InvalidTransitionException
7. Update entity.currentState with new SM state
8. Save entity
9. Call StateHistoryService.record() to create history

Key patterns from research:
- Use sm.hasStateChanged() to verify transition occurred
- Handle atomic persistence + SM transition
- Use @Transactional for atomicity
</action>
